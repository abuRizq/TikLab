# TikLab

**Virtual MikroTik RouterOS environment for development and testing** — simple commands, real behavior.

TikLab provides a Docker-based sandbox with real RouterOS v7 CHR and simulated users generating realistic network traffic. Ideal for testing billing systems, network automation, and MikroTik integrations.

---

## Prerequisites

- **Docker** installed and running
- **Go 1.22+** (for building from source only)

---

## Quick Install

### From source

```bash
git clone https://github.com/tiklab/tiklab.git
cd tiklab
make build
```

Then copy the binary to your PATH:

```bash
# Linux/macOS
sudo cp tiklab /usr/local/bin/

# Windows
copy tiklab.exe %USERPROFILE%\bin\
```

### Cross-platform build

```bash
make dist
# Output in dist/tiklab-{os}-{arch}
```

---

## Quick Start

```bash
# 1. Create the environment
tiklab create

# 2. Start RouterOS and simulated user traffic
tiklab start
```

Wait for `Ready.` — the sandbox is running with ~50 simulated users.

### Connect

| Service | Command / Address |
|---------|-------------------|
| **SSH** | `ssh admin@localhost -p 2222` (no password) |
| **RouterOS API** | `localhost:8728` |
| **Winbox** | `localhost:8291` |

---

## CLI Commands

| Command | Description |
|---------|-------------|
| `tiklab create` | Provision the environment (does not start it) |
| `tiklab start` | Boot RouterOS, apply config, start simulated traffic |
| `tiklab scale <count>` | Adjust simulated user count (1–500) |
| `tiklab reset` | Restore environment to original state without recreating the container |
| `tiklab destroy` | Remove the sandbox entirely |

### Examples

```bash
# Increase users for load testing
tiklab scale 200

# Return to clean state
tiklab reset

# Remove everything
tiklab destroy
```

## Simulated Users

~50 simulated users are generated by default with this behavior distribution:

| Profile | Share | Activity |
|---------|-------|----------|
| **idle** | ~40% | Ping and DNS only to keep session alive |
| **standard** | ~45% | Web browsing and small downloads |
| **heavy** | ~15% | Sustained high traffic (streaming/large downloads) |

Use `tiklab scale` to change the count dynamically without restarting the environment.

## Project Structure

```
cmd/
├── tiklab/          # Main CLI
└── tiklab-engine/   # Behavior engine (runs inside container)

internal/
├── cli/             # create, start, scale, reset, destroy commands
├── sandbox/         # State and lifecycle management
├── docker/          # Docker client interaction
├── routeros/        # RouterOS API client and config
└── engine/          # User simulation and traffic generation

build/
├── Dockerfile       # Container image
└── scripts/        # Entrypoint scripts
```
